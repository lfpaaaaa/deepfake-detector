
## Key Sequence Patterns (v3.1 Updates)

### 1. **Authentication Flow**
- Enhanced password policy validation (client + server)
- Client-side token validation before API calls âœ¨
- Expired/invalid tokens redirect to login
- User data attached to request context
- Registration link on login page âœ¨

### 2. **Job Lifecycle**
1. Create job â†’ Pending status (with `created_at` timestamp âœ¨)
2. Process detection â†’ Processing status
3. Validate results â†’ NaN/null checks âœ¨
4. Save results â†’ Completed status (sorted by timestamp âœ¨)
5. Generate reports â†’ Available for download (optimized heatmaps âœ¨)
